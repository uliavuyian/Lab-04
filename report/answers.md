# LAB04: Higher-Order Functions, map/filter, Decorators

## 1. What is a higher-order function?
A higher-order function is a function that can take other functions as arguments or return a function. It helps to write more flexible code and not repeat the same things.

## 2. What is the difference between `map` and list comprehension?
Map applies a function to each element and returns a map object. List comprehension does the same but looks like a normal loop and can include conditions. The result is the same, only the syntax is different.

## 3. What is a decorator?
A decorator is a function that can change what another function does or add new features to it.

## 4. What is the difference between a simple decorator and a decorator with arguments?
A simple decorator just changes or extends a function. A decorator with arguments first takes its own parameters and then adds them to how the function works.

## 5. Why is caching useful?
Caching saves the results of previous calculations so they are not computed again. It is useful for recursive functions or heavy computations.
